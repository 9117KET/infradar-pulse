# InfraRadarAI User Guide

This guide describes the intended user workflows. It can later become the basis for public help documentation.

## Find projects

1. Open the Projects dashboard.
2. Filter by region, country, sector, stage, value, confidence or risk.
3. Use the map when geography matters.
4. Use Ask InfraRadarAI for natural-language searches when a filter combination is easier to describe than configure.

## Evaluate a project

1. Open the project detail page.
2. Check status, stage, value, confidence and risk.
3. Review the description, source URL and evidence records.
4. Inspect stakeholders, milestones and updates.
5. Treat low-confidence or unverified records as leads that need more validation.

## Build a portfolio

1. Track projects that match the user’s market, region or account focus.
2. Add notes explaining why the project matters.
3. Revisit the Portfolio page to review saved projects.
4. Use Portfolio Chat to ask cross-project questions such as risk concentration, near-term tender movement or funding gaps.

## Monitor alerts

1. Open Alerts to see market, construction, regulatory, risk, security and financial signals.
2. Create alert rules for regions, sectors or project types.
3. Use alerts as triggers for outreach, diligence or internal briefings.

## Use intelligence summaries

1. Choose a report scope such as country, sector, tender outlook, portfolio risk or market snapshot.
2. Generate the report from current project and alert data.
3. Review citations and source context before sharing externally.
4. Export when the report is ready for team use.

## Researcher workflow

1. Review newly discovered or low-confidence projects in the Review Queue.
2. Confirm source URLs and evidence quality.
3. Approve, reject or request more research with a reason.
4. Use agent monitoring to pause, resume or inspect automated agents.
5. Preserve the rule that source URLs are required and unverified projects remain capped at low confidence.

## Project manager workflow

1. Track active projects in a portfolio.
2. Use project detail pages for status, milestones, risks, updates and contacts.
3. Watch alert changes for schedule, funding, regulatory or stakeholder issues.
4. Use reports and portfolio chat for internal status summaries.
